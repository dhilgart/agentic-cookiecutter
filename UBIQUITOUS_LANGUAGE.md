# Ubiquitous Language — agentic-cookiecutter

Terms used in developing and maintaining this cookiecutter. These are
forge-level terms and do NOT propagate to children.

---

## Terms

**Forge** — This entire repo (`agentic-cookiecutter/`). The factory that
includes all tooling: `cookiecutter.json`, `hooks/`, `docs/`, `tests/`,
`.claude/`, and the template. Changes to the forge affect how generation works.

**Template** — The `{{cookiecutter.project_slug}}/` directory. The pattern
that gets stamped out during generation. Changes to the template affect what
children look like.

**Child** — A repo generated by running `cookiecutter .` against the forge.
Has no ongoing relationship to the forge (unlike cruft, which tracks drift).

**Generation** — The act of running `cookiecutter .` to produce a child
from the template. Jinja2 variables are substituted at generation time.

**Drift** — When a child diverges from the current template.
Measured with `cruft check`.

**Skill** — A markdown file in `.claude/skills/` that encodes a repeatable
workflow for Claude Code. Lives in the template; symlinked into the forge.

**Hook script** — A Python file in `hooks/` that runs before or after
generation (`pre_gen_project.py`, `post_gen_project.py`).
These are cookiecutter hooks, NOT Claude Code hooks.

**Claude Code hook** — An entry in `.claude/hooks.json` that triggers a
shell command on Claude Code events (PostToolUse, Stop, PreToolUse).
Distinct from hook scripts.

**Quality gate** — An automated check that must pass before a change is
accepted. Tiers: deterministic (tools) and stochastic (LLM-judged).

**Opt-in feature** — A cookiecutter variable that conditionally includes
content in the child (e.g. `use_hypothesis`). Controlled via
Jinja2 conditionals.

---

## Anti-vocabulary

- ~~plugin~~ when referring to skills → use **skill**
- ~~template variables~~ → use **cookiecutter variables**
- ~~child project~~ / ~~child repo~~ → use **child**
- ~~parent repo~~ / ~~cookiecutter repo~~ → use **forge**
- ~~template scope~~ → use **template**
- ~~cookiecutter scope~~ → use **forge** (minus the template)
